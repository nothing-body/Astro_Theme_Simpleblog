# 部屬前須知

這份文件給第一次使用此 Astro 部落格模板的新手閱讀。公開版已移除私人文章、金鑰、GA4 ID 與個人網域，並保留三語範例文章，下載後可以立即安裝、build、預覽與部署。

## 1. 先確認環境

建議使用 Node.js 22.12.0 或更新版本。

```bash
node --version
pnpm --version
npm --version
```

本專案主要建議使用 pnpm，但部署與升級腳本也支援 npm。

如果沒有 pnpm，可先啟用 Corepack：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

## 2. 安裝與本機檢查

```bash
pnpm install
pnpm check
pnpm lint
pnpm build
```

npm 也可以：

```bash
npm install
npm run check
npm run lint
npm run build
```

`pnpm build` / `npm run build` 會產生 `dist/`。直接部署腳本會先執行 package.json 的 `build` script，再上傳 build 後的輸出資料夾。

## 3. 公開版範例文章

公開版保留三篇同 slug 的範例文章，避免空文章集合造成首頁、文章列表、分類、標籤或 sitemap 出錯：

- `src/content/blog/zh-tw/getting-started.md`
- `src/content/blog/en/getting-started.md`
- `src/content/blog/zh-cn/getting-started.md`

這些文章是正式文章，`draft: false`。你可以直接改內容，也可以新增自己的文章後刪除範例文章。

### 首頁書籤區

首頁「最新文章」下方的書籤區由 `src/components/BookmarkLinks.astro` 管理，三個語系首頁都會引用這個元件。

書籤資料集中在 `bookmarkRows` 陣列中，每一個內層陣列代表畫面上的一行：

```ts
const bookmarkRows = [
  [{ label: 'GitHub', href: 'https://github.com/' }],
  [
    { label: 'nodeseek', href: 'https://www.nodeseek.com/' },
    { label: 'LowEndTalk', href: 'https://lowendtalk.com/' },
  ],
];
```

新增書籤：在想放置的那一行加入一個物件，例如：

```ts
{ label: 'ipaddress', href: 'https://ipaddress.my/' },
```

移除書籤：刪除對應的 `{ label, href }` 物件即可。

增加更多行：在 `bookmarkRows` 最外層陣列新增一個新的內層陣列，例如：

```ts
[
  { label: 'Example', href: 'https://example.com/' },
],
```

注意事項：

- 外部連結會使用 `target="_blank"` 與 `rel="noopener noreferrer"`，避免新分頁取得原頁面的 `window.opener`。
- 請確認新增的網址使用 `https://`，並避免放入不可信、會重新導向到惡意頁面的連結。
- 修改後請執行 `pnpm check`、`pnpm lint`、`pnpm build`，確認元件與首頁都能正常建置。

## 4. 必改設定

請先複製環境檔案：

```bash
cp .env.example .env
cp .env.cloudflare.example .env.cloudflare
cp .env.vps.example .env.vps
cp .env.vercel.example .env.vercel
```

Windows PowerShell 可用：

```powershell
Copy-Item .env.example .env
Copy-Item .env.cloudflare.example .env.cloudflare
Copy-Item .env.vps.example .env.vps
Copy-Item .env.vercel.example .env.vercel
```

請至少設定：

```env
PUBLIC_SITE_URL=https://example.com
```

正式上線時請改成自己的網域。

## 5. Cloudflare Pages

`.env.cloudflare` 常用設定：

```env
CLOUDFLARE_API_TOKEN=your_cloudflare_api_token
CLOUDFLARE_ACCOUNT_ID=your_cloudflare_account_id
CLOUDFLARE_PAGES_PROJECT_NAME=your-pages-project-name
PUBLIC_SITE_URL=https://example.com
```

注意：

- `CLOUDFLARE_PAGES_PROJECT_NAME` 是 Cloudflare Pages project name。
- 如果帳號內沒有此 Pages project，腳本會自動建立，成功後寫回 `.env.cloudflare`，再繼續部署。
- 請檢查 `public/_headers`，確認 CSP、第三方資源、analytics 網域符合你的實際網站。

直接部署：

```bash
pnpm deploy:cf:only
npm run deploy:cf:only
```

## 6. VPS

`.env.vps` 常用設定：

```env
VPS_HOST=xxx.xxx.xxx
VPS_PORT=22
VPS_USER=deploy
VPS_TARGET_DIR=/var/www/example.com
VPS_SSH_KEY_PATH=~/.ssh/id_ed25519
VPS_SSH_PASSPHRASE=your_private_key_passphrase
```

說明：

- `VPS_USER` 是本機上傳 VPS 時執行 SSH/rsync 的使用者，可以是 `root`，也可以是 `deploy`、`ubuntu` 等非 root 使用者。
- 若 SSH 私鑰有密碼，建議優先使用 `ssh-agent`。
- `VPS_SSH_PASSPHRASE` 只能放在本機 `.env.vps` 或 CI/CD secrets，不要提交到 Git。
- 若 `VPS_USER` 不是 root 且不能寫入 `/var/www/...`，可先把 `VPS_TARGET_DIR` 設為 `/home/<user>/site-dist`。但上傳到暫存目錄不代表網站已更新，請自行在 VPS 上把 dist 內容移動或同步到 Web server 實際指向的資料夾。

直接部署：

```bash
pnpm deploy:vps:only
npm run deploy:vps:only
```

## 7. Vercel

`.env.vercel` 常用設定：

```env
VERCEL_TOKEN=your_vercel_token
VERCEL_ORG_ID=your_org_or_user_id
VERCEL_PROJECT_ID=your_project_id
```

建議設定 `VERCEL_ORG_ID` 與 `VERCEL_PROJECT_ID`，降低部署到錯誤 Vercel 專案的風險。

直接部署：

```bash
pnpm deploy:vercel:only
npm run deploy:vercel:only
```

## 8. 部署選單與雙語輸出

互動式選單：

```bash
pnpm deploy:menu
npm run deploy:menu
```

啟動後會先選擇語言，再進入部署主選單。也可以直接指定語言：

```bash
pnpm deploy:menu -- --lang=zh-tw
pnpm deploy:menu -- --lang=en
npm run deploy:menu -- --lang=en
```

非互動部署也支援語言：

```bash
pnpm deploy:switch --mode=direct:cf --lang=zh-tw
pnpm deploy:switch --mode=direct:vps --lang=en
DEPLOY_LANG=en pnpm deploy:cf:only
```

## 9. 多平台部署組合

常用指令：

```bash
pnpm deploy:cf:only
pnpm deploy:vps:only
pnpm deploy:vercel:only
pnpm deploy:cf:vps
pnpm deploy:cf:vercel
pnpm deploy:vps:vercel
pnpm deploy:all
```

模擬執行，不真的部署：

```bash
pnpm deploy:switch --mode=direct:cf+vps --dry-run
pnpm deploy:switch --mode=direct:cf+vps+vercel --dry-run --lang=en
```

## 10. GitHub / GitLab / Codeberg CI/CD

Git provider 模式只 push 原始碼，真正 build 與部署應在 CI/CD 內執行。

需要先建立對應檔案：

- GitHub: `.github/workflows/deploy.yml`
- GitLab: `.gitlab-ci.yml`
- Codeberg/Woodpecker: `.woodpecker.yml`

也要在平台內設定 secrets，例如 Cloudflare token、VPS SSH key、Vercel token。

## 11. scripts 腳本作用

- `deploy_menu.mjs`：互動式部署選單，會先選語言。
- `deploy_switch.mjs`：命令列部署切換器，適合 CI/CD 或直接指定模式。
- `deploy_i18n.mjs`：部署腳本共用 i18n 字典，支援繁中與英文。
- `deploy_lib.mjs`：部署模式、部署組合與指令生成邏輯。
- `deploy_runtime.mjs`：Node.js、pnpm、npm 與跨平台 runner 偵測。
- `deploy_safety.mjs`：部署前 `.gitignore` 與敏感檔案安全檢查。
- `uploaddist_cf.mjs`：build 後上傳輸出資料夾到 Cloudflare Pages。
- `uploaddist_vps.mjs`：build 後透過 SSH/rsync 上傳輸出資料夾到 VPS。
- `uploaddist_vercel.mjs`：透過 Vercel CLI build/deploy。
- `upgrade_astro.mjs`：安全升級 Astro 相關套件。
- `analysis.mjs`：專案自檢腳本。
- `run-e2e.mjs`：端對端測試入口。

## 12. 安全升級 Astro

雙語、跨平台、支援 pnpm/npm：

```bash
pnpm upgrade:astro -- --lang=zh-tw --dry-run
pnpm upgrade:astro -- --lang=zh-tw --dry-run --clean-install
pnpm upgrade:astro -- --lang=en
npm run upgrade:astro -- --lang=en --yes
```

安全設計：

- 雙語是指終端機輸出語言。`--lang=zh-tw` 會顯示繁體中文提示、警告、確認與錯誤訊息；`--lang=en` 會顯示英文訊息。
- `--lang` 不會改變實際升級哪些套件、不會改變網站內容語言，也不是另一套升級流程。
- `--clean-install` 會在正式升級前刪除可再生目錄：`node_modules`、`.astro`、`dist`，再執行正常升級與驗證流程。
- 腳本不會自動刪除 lockfile，因為 lockfile 能保留可重現安裝，也讓升級失敗時更容易檢查與回復。
- 自動偵測 `package.json` 內 Astro 相關套件，例如 `astro`、`@astrojs/*`、`astro-*`、`prettier-plugin-astro`、`astro-eslint-parser`。
- 預設要求 git working tree 乾淨，避免升級後和既有變更混在一起難以回復。
- 如確定要在未提交狀態升級，可加入 `--allow-dirty`。
- 支援 `--dry-run`，只列出將執行的升級與驗證指令，不修改任何檔案。
- 升級後使用 package.json 既有 `check`、`lint`、`build` scripts 驗證，不直接把流程寫死成特定 Astro CLI 指令，降低耦合。

常用參數：

| 參數 | 說明 |
| --- | --- |
| `--lang=zh-tw` | 使用繁體中文輸出 |
| `--lang=en` | 使用英文輸出 |
| `--dry-run` | 只預覽，不修改依賴 |
| `--yes` | 跳過確認提示 |
| `--allow-dirty` | 允許在 git working tree 非乾淨時升級 |
| `--clean-install` | 升級前刪除 `node_modules`、`.astro`、`dist` |
| `--skip-check` | 跳過 `pnpm check` / `npm run check` |
| `--skip-lint` | 跳過 `pnpm lint` / `npm run lint` |
| `--skip-build` | 跳過 `pnpm build` / `npm run build` |

## 13. 上傳前安全檢查

`.gitignore` 必須忽略：

- `.env`
- `.env.cloudflare`
- `.env.vps`
- `.env.vercel`
- `.dev.vars`
- `.wrangler/`
- `.vercel/`
- `.ssh/`
- `*.pem`
- `*.key`
- `node_modules/`
- `dist/`
- `.astro/`

部署腳本會在部署前檢查 `.gitignore`。若缺少敏感規則，會詢問是否自動修補。

## 14. 公開上傳建議

真正建議上傳 GitHub 的資料夾是：

```text
astro-multilingual-deploy-template
```

不要上傳：

- `node_modules/`
- `dist/`
- `.astro/`
- `.pnpm-store/`
- `.env*` 真實檔案
- SSH 私鑰或任何 token
